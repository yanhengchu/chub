- [x] 对明显过短的 Token 给出安全警告，并在 README 提供随机 Token 生成方式。
- [x] 实现节点和系统状态采集服务。
- [x] 实现受保护的状态接口。
- [x] 统一成功与错误响应结构。
- [x] 补充认证、状态采集和状态接口测试。

验收：有效 Token 可读取完整状态；缺失或无效 Token 返回带 `WWW-Authenticate: Bearer` 的 `401`；服务端未配置 Token 时受保护接口返回 `503`；健康检查始终可公开访问；响应、控制台日志和文件日志均不包含 Token。

## 5. M4：日志查看

- [x] 实现只读取配置指定活动日志的尾部读取服务。
- [x] 限制默认/最大行数、总读取量和单行长度。
- [x] 拒绝文件路径及其他额外查询参数。
- [x] 对当前 Token 和明显的 Bearer 凭证做返回前脱敏。
- [x] 实现受保护的日志接口。
- [x] 统一空文件、文件暂时不存在和读取失败的响应语义。
- [x] 验证日志写入和接口响应不包含访问 Token。
- [x] 补充尾部顺序、轮转边界、大小限制、认证和接口安全测试。

验收：正确 Token 可按旧到新顺序读取当前活动日志最后 N 行；用户不能通过任何参数指定文件；行数、总读取量和单行长度受限；空文件和暂时不存在返回空列表；权限或读取错误返回受控 `503`；当前 Token 和 Bearer 凭证在返回前被脱敏；不读取轮转文件；日志读取失败不影响健康、状态和任务接口。
