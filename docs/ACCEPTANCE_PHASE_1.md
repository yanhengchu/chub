## 5. 双节点与隔离结果

- 手机通过可信网络可分别访问 MacBook 与 Ubuntu 节点。
- 任一节点停止时，另一节点的页面、状态、任务和日志不受影响。
- 两个节点使用各自独立的 Token 和 `settings.local.yaml`。
- 恢复节点后可重新正常访问，不依赖另一节点的运行状态。

## 6. 遗留与后续优化

- 第一阶段仍以可信局域网或虚拟组网为部署边界，不直接暴露公网。
- 浏览器端目前以源码约束和人工手机验收为主；后续可增加 Playwright 端到端测试。
- 依赖使用兼容版本范围；后续正式发布可增加跨平台锁定文件。
- 应用版本同时存在于包元数据和节点配置；下一阶段可收敛为单一版本来源。
- 独立发布目录、自动升级和回滚流程留待下一阶段设计。
